üîß UTF Encoding Forms

UTF-8 (8-bit Unicode Transformation Format)

Variable-width encoding that uses 1 to 4 bytes per character

// UTF-8 encoding examples
'A' = 41 (1 byte)      - ASCII compatible
'√±' = C3 B1 (2 bytes)  - Latin-1 Supplement
'‰∏≠' = E4 B8 AD (3 bytes) - CJK
'üòÄ' = F0 9F 98 80 (4 bytes) - Emoji

// UTF-8 encoding rules
0xxxxxxx: 1 byte (ASCII)
110xxxxx 10xxxxxx: 2 bytes
1110xxxx 10xxxxxx 10xxxxxx: 3 bytes
11110xxx 10xxxxxx 10xxxxxx 10xxxxxx: 4 bytes

UTF-16 (16-bit Unicode Transformation Format)

Variable-width encoding using 16-bit code units

// UTF-16 encoding examples
'A' = 0041 (1 code unit)
'√±' = 00F1 (1 code unit)
'‰∏≠' = 4E2D (1 code unit)
'üòÄ' = D83D DE00 (2 code units - surrogate pair)

// Surrogate pairs for supplementary characters
High surrogate: D800-DBFF
Low surrogate: DC00-DFFF
Code point = (high - D800) √ó 400 + (low - DC00) + 10000

UTF-32 (32-bit Unicode Transformation Format)

Fixed-width encoding using 32-bit code units

// UTF-32 encoding examples
'A' = 00000041 (4 bytes)
'√±' = 000000F1 (4 bytes)
'‰∏≠' = 00004E2D (4 bytes)
'üòÄ' = 0001F600 (4 bytes)

// Direct code point representation
// Each character uses exactly 4 bytes
// No encoding/decoding complexity

// Script detection
function detectScript(text) {
    const scripts = {
        'Latin': /[\u0000-\u007F\u0080-\u00FF\u0100-\u017F]/,
        'Greek': /[\u0370-\u03FF\u1F00-\u1FFF]/,
        'Cyrillic': /[\u0400-\u04FF\u0500-\u052F]/,
        'Han': /[\u4E00-\u9FFF\u3400-\u4DBF]/,
        'Hiragana': /[\u3040-\u309F]/,
        'Katakana': /[\u30A0-\u30FF]/,
        'Hangul': /[\uAC00-\uD7AF\u1100-\u11FF]/,
        'Arabic': /[\u0600-\u06FF\u0750-\u077F]/,
        'Devanagari': /[\u0900-\u097F]/,
        'Thai': /[\u0E00-\u0E7F]/,
        'Hebrew': /[\u0590-\u05FF]/
    };
    
    const detected = [];
    for (const [script, regex] of Object.entries(scripts)) {
        if (regex.test(text)) {
            detected.push(script);
        }
    }
    return detected;
}

‚úÖ Best Practices

String Operations

  • Use Array.from() or [...str] for character iteration
  • Avoid str.length for multi-byte characters
  • Use codePointAt() instead of charCodeAt()
  • Normalize strings before comparison

Encoding/Decoding

  • Always specify encoding when reading/writing files
  • Use UTF-8 as the default encoding
  • Handle encoding errors gracefully
  • Validate input for proper Unicode sequences

Database and Storage

  • Use UTF-8 encoding for database columns
  • Set proper collation for sorting
  • Consider normalization for search operations
  • Validate data before storage
