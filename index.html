Design Patterns

Singleton Pattern

Ensures a class has only one instance and provides global access to it.

class Singleton {
    private static instance: Singleton;
    private constructor() {}
    
    public static getInstance(): Singleton {
        if (!Singleton.instance) {
            Singleton.instance = new Singleton();
        }
        return Singleton.instance;
    }
}

Factory Pattern

Creates objects without specifying their exact classes.

interface Animal {
    makeSound(): void;
}

class Dog implements Animal {
    makeSound() { console.log('Woof!'); }
}

class Cat implements Animal {
    makeSound() { console.log('Meow!'); }
}

Object-Oriented Programming

Master the fundamentals of OOP

Encapsulation

Bundling data and methods that operate on that data within a single unit.

class BankAccount {
    private balance: number;
    
    constructor(initialBalance: number) {
        this.balance = initialBalance;
    }
    
    public getBalance(): number {
        return this.balance;
    }
    
    public deposit(amount: number): void {
        if (amount > 0) {
            this.balance += amount;
        }
    }
}

Inheritance

Creating new classes that are built upon existing classes.

class Animal {
    protected name: string;
    
    constructor(name: string) {
        this.name = name;
    }
    
    public makeSound(): void {
        console.log('Some sound');
    }
}

class Dog extends Animal {
    public makeSound(): void {
        console.log('Woof!');
    }
}

Polymorphism

Ability to present the same interface for different underlying forms.

interface Shape {
    calculateArea(): number;
}

class Circle implements Shape {
    constructor(private radius: number) {}
    
    calculateArea(): number {
        return Math.PI * this.radius ** 2;
    }
}

class Rectangle implements Shape {
    constructor(private width: number, private height: number) {}
    
    calculateArea(): number {
        return this.width * this.height;
    }
}
