üìÑ HTML (HyperText Markup Language)

The standard markup language for creating web pages and web applications

Language Overview

HTML is the backbone of the World Wide Web. It provides the structure and content for web pages, allowing developers to create documents that can be displayed in web browsers. HTML uses a system of tags and attributes to define the structure and content of web pages.

First Released 1993
Latest Version HTML5
Paradigm Markup
Typing None

üîë Key Features

Tag-Based Structure

Uses opening and closing tags to define elements and their content

Hierarchical Structure

Elements can be nested within other elements to create complex structures

Semantic Elements

HTML5 introduced semantic tags that provide meaning to content structure

Responsive Design

Supports responsive design principles through meta tags and flexible layouts

üìö Basic Syntax

HTML Document Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
</head>
<body>
    <header>
        <h1>Welcome to My Website</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <section id="home">
            <h2>Home Section</h2>
            <p>This is the main content area.</p>
        </section>
    </main>
    
    <footer>
        <p>© 2024 My Website</p>
    </footer>
</body>
</html>

üéØ HTML5 Semantic Elements

<header>

Defines a header for a document or section

<header>
    <h1>Website Title</h1>
    <nav>Navigation menu</nav>
</header>

<nav>

Defines a set of navigation links

    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
    </ul>

<main>

Specifies the main content of a document

    <p>Main content goes here...</p>
</main>

<section>

Defines a section in a document

<section>
    <h2>Section Heading</h2>
    <p>Section content...</p>
</section>

üîß HTML Attributes

  • class - CSS class names
  • style - Inline CSS styles

Form Attributes

  • type - Input type (text, email, password)
  • name - Form field name
  • value - Default value
  • placeholder - Hint text
  • required - Required field
  • disabled - Disabled field

Link Attributes

  • href - URL destination
  • target - Link target (_blank, _self)
  • rel - Relationship to current page
  • download - Download link
  • title - Link description

üöÄ Best Practices

1. Semantic HTML

Use semantic elements to provide meaning to your content structure. This improves accessibility and SEO.

2. Proper Document Structure

Always include DOCTYPE, html, head, and body tags. Use proper heading hierarchy (h1, h2, h3, etc.).

3. Accessibility

Include alt attributes for images, use proper form labels, and ensure keyboard navigation works.

4. Clean Code

Use consistent indentation, meaningful names for IDs and classes, and comment complex sections.

5. Validation

Validate your HTML using the W3C validator to ensure standards compliance.
